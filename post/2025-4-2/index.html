enveloppe的部署中,可能会出现一定的问题[Bug]: [META] Error during publishing, mostly because of GitHub REST API. · Issue #389 · Enveloppe/obsidian-enveloppe 这个bug是尚且没有修复的 其来源有些玄学 本人被其整破防了 但是最后通过重新安装该插件,问题就得到了解决 所以如果你不幸遇到了这个问题,请不要着急,重新安装即可!

github模块

我们直接写出来的markdown文件夹是不能够直接作为网页进行渲染的,因此,hugo在其外层文件夹中,通过一系列的编码程序,将你的markdown文档转化为了可以部署为静态网页的html文件

而这些可以部署的内容就是public文件夹

可以得知我们的博客本身的部署,其实是不需要用到外面的hugo主文件夹的,其作用就是帮助我们进行更新.

因此,hugo主文件夹创建之后,会接受两个指令

  • 一为enveloppe所传输的更新博客
  • 一为运行hugo -D指令,进行网站更新部署的指令

而hugo -D的指令则传输到了public文件夹的github库中(确保该库是公开的). 由于该库名字为用户名.github.io,是你默认的githubpage的基础域名(如果你创建了其他部署在github上的网页,也会默认将网页部署在这个域名下) 因此它一经过更新,就会将你的静态网页部署到github的网页服务上.

flow

  • obsidian书写博客
  • enveloppe自动转化博客格式,将其上传到hugo主文件夹的库中
  • 主文件夹库在接受到上传后,自动运行hugo-D
  • 运行hugo -D后的文件传输至public库中
  • pubic库自动重新部署
